## AI Reservation Helper

**AI Reservation Helper** — это небольшой сервис‑помощник для администраторов компьютерного клуба, который:

- **отслеживает звонки** через облачную АТС (Beeline Cloud PBX),
- **скачивает записи разговоров** и транскрибирует их в текст через **OpenAI Whisper**,
- на основе транскрипции генерирует **готовый шаблон сообщения о бронировании** (GPT‑4),
- позволяет администратору **скорректировать текст и отправить его гостю в WhatsApp** через **Green‑API**,
- а также умеет генерировать тот же шаблон **по текстовой переписке с гостем**, если админ просто вставляет диалог в Telegram‑бот.

Проект изначально делался под клуб Lockers, но легко адаптируется под другое заведение.

---

### Архитектура

- **`main.py`**
  - Flask‑приложение с вебхуком `/subscription` для событий из облачной АТС Beeline.
  - Фоновый поток, который:
    - ищет новые звонки в БД,
    - скачивает запись разговора,
    - отправляет аудио в OpenAI Whisper,
    - на основе транскрипции формирует текст сообщения (GPT‑4),
    - сохраняет сгенерированный текст в БД.
  - Telegram‑бот (через `pyTelegramBotAPI`):
    - команда `/start` — описание возможностей,
    - команда `/menu` — админ‑меню,
    - простое сообщение от админа — генерация шаблона по вставленной переписке,
    - inline‑кнопки:
      - обновление подписки на звонки,
      - включение/выключение анализа,
      - получение кода для авторизации WhatsApp,
      - «Отправить», «Корректировать» и «Удалить сообщение».
- **SQLite‑БД (`db.db`)**
  - **`settings`** — настройки системы:
    - `token` — токен подписки на звонки в Beeline,
    - `status_analyze` — включён ли анализ (0/1),
    - `last_time_got_token` — когда последний раз обновляли подписку.
  - **`recordings`** — звонки и сообщения:
    - `phone_number` — номер гостя,
    - `ext_tracking_id` — идентификатор записи звонка,
    - `transcript` — транскрипция разговора,
    - `generated_message` — текст, который уходит гостю,
    - `sent_to_user` — флаг отправки.

База данных и структура таблиц создаются **автоматически при первом запуске**, вручную файл `db.db` в репозиторий добавлять не нужно.

### Установка и запуск

1. **Склонируйте репозиторий и перейдите в папку проекта**

```bash
git clone https://github.com/<your-username>/ai_reservation_helper.git
cd ai_reservation_helper
```

2. **Создайте и активируйте виртуальное окружение (рекомендуется)**

```bash
python -m venv venv
venv\Scripts\activate  # Windows
```

3. **Установите зависимости**

```bash
pip install -r requirements.txt
```

4. **Заполните настройки в `main.py`**

В начале файла задайте свои значения:

- `ATS_TOKEN` — токен для Cloud PBX Beeline;
- `phone_ats` — номер внутреннего аккаунта в АТС;
- `OPENAI_API_KEY` — API‑ключ OpenAI (для GPT‑4 и Whisper);
- `TELEGRAM_TOKEN` — токен Telegram‑бота;
- `apiUrlInstance`, `idInstance`, `apiTokenInstance`, `phoneNumber` — данные от Green‑API (WhatsApp);
- `ADMIN_ID` — ваш Telegram user ID (чтобы только вы имели доступ к меню);
- `HOST` — домен или IP, по которому Flask‑приложение доступно снаружи (нужен для вебхука Beeline), например: `"example.com"` или `"1.2.3.4"`.

5. **Запустите приложение**

```bash
python main.py
```

При первом запуске:

- будет создан файл `db.db` c нужными таблицами,
- создастся папка `records/` для хранения записей звонков.

Flask‑сервер поднимется на порту `80`, Telegram‑бот и обработка транскрипций стартуют в фоновых потоках.

### Настройка интеграций

- **Beeline Cloud PBX**
  - Настройте подписку на события звонков (ADVANCED_CALL) так, чтобы вебхук указывал на:
    - `http://<HOST>/subscription`
  - Токен авторизации указывается в `ATS_TOKEN`.

- **OpenAI**
  - Получите API‑ключ в личном кабинете OpenAI.
  - Укажите его в `OPENAI_API_KEY`.
  - В проекте используется:
    - `gpt-4` — для генерации текста сообщений,
    - `whisper-1` — для транскрибации аудио.

- **Green‑API (WhatsApp неофициальный)**
  - Зарегистрируйтесь в Green‑API и получите:
    - URL инстанса,
    - `idInstance`,
    - `apiTokenInstance`.
  - Заполните соответствующие константы в `main.py`.

- **Telegram‑бот**
  - Создайте бота через `@BotFather`, получите токен и пропишите в `TELEGRAM_TOKEN`.
  - Узнайте свой `user_id` (например, через бота `@userinfobot`) и пропишите в `ADMIN_ID`.

### Как это выглядит в работе

- Администратор принимает звонок от гостя, обсуждает детали брони.
- После завершения звонка:
  - Beeline Cloud PBX шлёт вебхук в Flask,
  - сервис фиксирует `extTrackingId` и номер телефона,
  - фоновый поток скачивает запись, расшифровывает её и генерирует сообщение по шаблону.
- В Telegram‑боте администратор видит:
  - текст сообщения с деталями бронирования,
  - кнопки:
    - **«Отправить»** — отправка гостю в WhatsApp,
    - **«Корректировать»** — правка текста и повторное сохранение,
    - **«Удалить сообщение»**.
- Также админ может:
  - вставить переписку с гостем (из любого мессенджера) **одним сообщением** в бота,
  - получить готовый шаблон уведомления о брони по этой переписке.
